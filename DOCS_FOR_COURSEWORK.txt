Актуальность темы. В современных условиях информационные системы подвергаются
постоянным атакам со стороны злоумышленников. По данным OWASP Top 10 2021,
веб-приложения сталкиваются с такими угрозами как:

- SQL Injection (SQLi)
- Cross-Site Scripting (XSS)
- Broken Access Control
- Security Misconfiguration

Традиционные методы обнаружения атак, основанные на сигнатурах, имеют
ограниченную эффективность против новых типов атак. Системы на основе
машинного обучения способны обнаруживать ранее неизвестные угрозы.

Цель работы: Разработка системы обнаружения веб-атак ORCHID на основе
ансамбля моделей машинного обучения с возможностью детектирования
многоступенчатых эксплойтов.

Задачи:
1. Исследовать существующие методы обнаружения веб-атак
2. Разработать математические модели классификации атак
3. Реализовать систему сбора и анализа трафика
4. Создать веб-интерфейс для мониторинга
5. Провести тестирование системы

Объект исследования: Процессы обнаружения веб-атак в реальном времени.

Предмет исследования: Алгоритмы машинного обучения для классификации
веб-атак.

4.1. Алгоритм Isolation Forest
--------------------------------------------------------------------------------

Isolation Forest - алгоритм обнаружения аномалий, основанный на принципе
изоляции наблюдений.

Математическая модель:

Пусть X = {x₁, x₂, ..., xₙ} - множество наблюдений в d-мерном пространстве.

Дерево изоляции строится рекурсивно:

1. Выбираем случайный признак q ∈ {1, 2, ..., d}
2. Выбираем случайное значение разделения p между min и max значениями q
3. Разделяем данные: X_left = {x ∈ X | x_q < p}, X_right = X \ X_left
4. Рекурсивно строим поддеревья

Формула оценки аномальности:

s(x, n) = 2^(-E(h(x))/c(n))

где:
- E(h(x)) - средняя глубина узла x по всем деревьям
- c(n) - средняя длина пути неудачного поиска в BST:

c(n) = 2H(n-1) - (2(n-1)/n)

H(i) = ln(i) + γ (гармоническое число)
γ ≈ 0.5772156649 (константа Эйлера-Маскерони)

Интерпретация:
- s(x, n) ≈ 1 → аномалия
- s(x, n) < 0.5 → норма
- s(x, n) ≈ 0.5 → неопределено

4.2. Алгоритм Random Forest
--------------------------------------------------------------------------------

Random Forest - ансамблевый метод классификации на основе решающих деревьев.

Математическая модель:

Пусть D = {(xᵢ, yᵢ)}ᵢ₌₁ⁿ - обучающая выборка, где xᵢ ∈ ℝᵈ, yᵢ ∈ {1, ..., K}

Строим B деревьев:

Для b = 1 до B:
  1. Формируем бутстрап-выборку D_b ⊂ D
  2. Строим дерево T_b на D_b со случайным выбором признаков
  3. Для каждого узла выбираем m случайных признаков из d

Формула классификации:

ŷ = argmax_k Σᵦ₌₁ᴵ I(T_b(x) = k)

где I(·) - индикаторная функция.

Важность признаков:

VI(j) = (1/B) Σᵦ₌₁ᴵ VI_b(j)

VI_b(j) = Σ_t∈T_b: split on j Δi(s_t, t)

где Δi - уменьшение неопределенности (Gini/Entropy).

4.3. Алгоритм Gradient Boosting
--------------------------------------------------------------------------------

Gradient Boosting - метод построения ансамбля слабых моделей.

Математическая модель:

Инициализация:
F₀(x) = argmin_γ Σᵢ₌₁ⁿ L(yᵢ, γ)

Для m = 1 до M:
  1. Вычисляем градиенты:
     rᵢₘ = -[∂L(yᵢ, F(xᵢ))/∂F(xᵢ)]_{F=Fₘ₋₁}
  
  2. Обучаем слабую модель hₘ на {(xᵢ, rᵢₘ)}
  
  3. Находим оптимальный шаг:
     γₘ = argmin_γ Σᵢ₌₁ⁿ L(yᵢ, Fₘ₋₁(xᵢ) + γhₘ(xᵢ))
  
  4. Обновляем:
     Fₘ(x) = Fₘ₋₁(x) + γₘhₘ(x)

Функция потерь (Log Loss):
L(y, p) = -[y·log(p) + (1-y)·log(1-p)]

4.4. Ансамбль моделей
--------------------------------------------------------------------------------

ORCHID использует взвешенное голосование моделей:

P(final) = w₁·P(IF) + w₂·P(RF) + w₃·P(GB)

где:
- P(IF) - вероятность от Isolation Forest
- P(RF) - вероятность от Random Forest
- P(GB) - вероятность от Gradient Boosting
- w₁ + w₂ + w₃ = 1

Веса определяются экспериментально:
- w₁ = 0.3 (Isolation Forest)
- w₂ = 0.4 (Random Forest)
- w₃ = 0.3 (Gradient Boosting)

Формула детектирования:

detect(x) = 1, если P(final) > θ
            0, иначе

где θ = 0.5 - порог срабатывания.

7.1. Метрики качества
--------------------------------------------------------------------------------

Accuracy (Точность):
ACC = (TP + TN) / (TP + TN + FP + FN)

Precision (Точность положительного класса):
P = TP / (TP + FP)

Recall (Полнота):
R = TP / (TP + FN)

F1-Score (Гармоническое среднее):
F1 = 2 · (P · R) / (P + R)

где:
- TP (True Positive) - верно обнаруженные атаки
- TN (True Negative) - верно пропущенный легитимный трафик
- FP (False Positive) - ложные срабатывания
- FN (False Negative) - пропущенные атаки
